Overview

Redesign the enterprise onboarding flow to align with the new design system tokens and improve conversion at each step. This task covers UI updates, copy revisions, and API integration for step 4.

Acceptance Criteria

All onboarding steps use DS tokens from the v2 token set
Step 3 CTA meets minimum tap-target size on mobile breakpoints
Copy on steps 4–6 reviewed and approved by content team
Step 4 calls POST /api/v3/onboarding/init and handles error states gracefully
Progress indicator is visually consistent across all steps
QA sign-off on Chrome, Safari and Firefox at 1280px and 1440px

Technical Notes

# Onboarding init endpoint

POST /api/v3/onboarding/init

Content-Type: application/json

{ "userId": "string", "plan": "enterprise" }

# Returns session token. Response < 120ms in staging.
