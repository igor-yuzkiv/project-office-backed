
# Task: Add Breadcrumb Support to Layout Header

Потрібно додати підтримку breadcrumbs у layout, схожим підходом як зараз реалізовані `headerActions`.

## Контекст

У проєкті вже є `layoutStore`, який використовується для керування layout state, зокрема для header actions.

Зараз у компоненті:

`resources/js/app/shell/ui/header/AppHeader.vue`

відображається title. Потрібно додати можливість замість title показувати breadcrumbs, якщо вони задані в layout state.

Для breadcrumbs потрібно використати PrimeVue компонент:

```ts
import Breadcrumb from 'primevue/breadcrumb'
````

## Що потрібно зробити

### 1. Оновити layout store

У `layoutStore` додати state для breadcrumbs.

Breadcrumb item має підтримувати мінімально такі поля:

```ts
type BreadcrumbItem = {
  label: string
  to?: RouteLocationRaw
}
```

Store має мати методи для встановлення і очищення breadcrumbs, аналогічно до header actions.

Бажано реалізувати scoped-підхід через `symbol`, як для header actions, щоб при unmount сторінки не було ризику випадково очистити breadcrumbs, які вже виставила інша сторінка.

Приклад очікуваного API на рівні store:

```ts
setBreadcrumbs(scopeId, items)
clearBreadcrumbs(scopeId)
```

Актуальні breadcrumbs мають бути доступні з layout store для використання в header.

---

### 2. Додати composable для breadcrumbs

Створити helper composable, аналогічний до composable для header actions.

Очікуваний приклад використання на сторінці:

```ts
useBreadcrumbs([
  {
    label: 'Projects',
    to: { name: 'projects.index' },
  },
  {
    label: project.name,
    to: {
      name: 'project-details.details',
      params: { projectId: project.id },
    },
  },
  {
    label: 'Tasks',
  },
])
```

Composable має:

- приймати масив breadcrumbs або getter;
    
- записувати breadcrumbs у `layoutStore`;
    
- автоматично очищати breadcrumbs при dispose/unmount;
    
- підтримувати reactive значення, щоб breadcrumbs оновлювались після завантаження даних.
    

Бажаний API:

```ts
useBreadcrumbs(items)
```

Де `items` може бути як звичайним масивом, так і функцією:

```ts
useBreadcrumbs(() => [
  {
    label: 'Projects',
    to: { name: 'projects.index' },
  },
  {
    label: project.value?.name ?? 'Project',
  },
])
```

---

### 3. Оновити AppHeader.vue

У компоненті:

`resources/js/app/shell/ui/header/AppHeader.vue`

додати відображення breadcrumbs.

Логіка:

- якщо в `layoutStore` задані breadcrumbs і масив не пустий — показувати PrimeVue `Breadcrumb`;
    
- якщо breadcrumbs немає — залишити поточну поведінку і показувати title, як зараз.
    

Тобто header має працювати так:

```text
breadcrumbs задані → показуємо Breadcrumb
breadcrumbs не задані → показуємо title
```

Поточну логіку title не ламати.

---

### 4. PrimeVue Breadcrumb item format

Потрібно перевірити формат, який очікує PrimeVue `Breadcrumb`.

Якщо наш внутрішній формат:

```ts
{
  label: string
  to?: RouteLocationRaw
}
```

не повністю підходить напряму, додати mapping у `AppHeader.vue` або computed property.

Не потрібно змінювати формат breadcrumbs у всьому layout store тільки через специфіку PrimeVue. Store має зберігати нормальний внутрішній формат, а UI-компонент може адаптувати його під PrimeVue.

---

## Acceptance Criteria

- У `layoutStore` є state для breadcrumbs.
    
- Є composable `useBreadcrumbs`, який працює аналогічно до `useHeaderActions`.
    
- Breadcrumbs автоматично очищаються при unmount/dispose сторінки.
    
- `AppHeader.vue` показує PrimeVue `Breadcrumb`, якщо breadcrumbs задані.
    
- Якщо breadcrumbs не задані, header показує title так само, як і раніше.
    
- Поточна логіка header actions не зламана.
    
- Рішення не має вимагати ручного `onMounted/onUnmounted` на кожній сторінці.
    
- Рішення має підтримувати reactive breadcrumbs після завантаження entity, наприклад project/task name.
